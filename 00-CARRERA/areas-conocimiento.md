# Áreas de conocimiento — derivación y validación

Las áreas se **derivan del perfil de egresado** y se **validan** contra dos estándares reales:
- **CS2023** (ACM/IEEE-CS/AAAI): desde 2013 el foco pasó de "qué se enseña" a "qué se
  aprende" (modelo de competencias). La IA subió a núcleo y la matemática se expandió.
- **OSSU** (Open Source Society University): currículum de CS armado con material abierto
  (MIT, Harvard, Princeton...), diseñado según los requisitos de una carrera de grado.

| # | Área (derivada del perfil) | Materias | Validación CS2023 | Validación OSSU |
|---|----------------------------|----------|-------------------|-----------------|
| 1 | Fundamentos de programación y software | PRG1, PRG2 | SDF, PL, SE | Intro CS, Core programming |
| 2 | Matemática para IA | MDIS, ALG, AMAT, PROB | MSF (expandida) | Core math |
| 3 | Algoritmos y estructuras de datos | ALGO | AL | Core theory |
| 4 | Sistemas de computación | FINF, SO, REDES | AR, OS, NC | Core systems |
| 5 | Datos | BDATOS | DM | Core applications |
| 6 | Inteligencia Artificial y ML | IIA, ML, DL, NLP, CV, RL, ROB | AI/ML (núcleo nuevo) | Core/Advanced applications |
| 7 | Ingeniería de software y productos | ISW, PROY | SE | Core applications + Final project |
| 8 | Ética, sociedad y profesión | ETIC | SEP (transversal) | Core ethics |
| 9 | Emprendimiento, liderazgo y comunicación | EMP, COM | — (fuera de alcance) | — (fuera de alcance) |

**Nota de honestidad curricular**: las áreas 1–8 heredan la validación de OSSU/CS2023.
El área 9 **no**: ambos estándares excluyen deliberadamente la formación general/no-CS.
Se incluye porque es central al perfil de UdeSA, pero sus fuentes son de otra naturaleza y
están marcadas con ⚠️ en la carpeta `fuentes/`.
