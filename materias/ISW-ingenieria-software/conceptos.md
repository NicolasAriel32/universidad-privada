# ISW — Ingeniería de Software

- **Nivel**: Transversal
- **Área de conocimiento**: Ingeniería de software y productos
- **Fuente canónica**: UBC 'Software Engineering: Introduction' + 'Software Engineering at Google'

## Objetivo de la materia

Construir software mantenible en equipo: control de versiones, testing, diseño y proceso.

## Temas centrales y objetivos de aprendizaje

### 1. Proceso y colaboración

**Conceptos que lo integran**: Git (branching, merging, PRs), code review, metodologías ágiles, especificación de requisitos.

**Objetivos de aprendizaje** (nivel Bloom entre paréntesis):
- Aplicar (Aplicar) un flujo de trabajo con branches y pull requests.
- Revisar (Evaluar) código de otros dando feedback específico y constructivo — conecta con COM.
- Traducir (Aplicar) un requisito ambiguo de usuario a una especificación verificable.

### 2. Calidad y testing

**Conceptos que lo integran**: tests unitarios, tests de integración, TDD (introducción), cobertura, refactorización.

**Objetivos de aprendizaje** (nivel Bloom entre paréntesis):
- Escribir (Aplicar) tests unitarios que cubran casos borde, no solo el camino feliz.
- Refactorizar (Aplicar) código con olor manteniendo los tests en verde.
- Evaluar (Evaluar) qué cobertura y qué tipo de tests necesita cada módulo según su riesgo.

### 3. Diseño de software

**Conceptos que lo integran**: principios de diseño (acoplamiento/cohesión), patrones comunes, APIs, arquitectura de aplicaciones, deuda técnica.

**Objetivos de aprendizaje** (nivel Bloom entre paréntesis):
- Diseñar (Crear) la estructura de módulos de una aplicación mediana justificando los límites.
- Detectar (Analizar) acoplamiento excesivo en un diseño existente.
- Decidir (Evaluar) cuándo pagar deuda técnica y cuándo es aceptable postergarla.

---
_Objetivos redactados con taxonomía de Bloom revisada (Recordar → Entender → Aplicar → Analizar → Evaluar → Crear) y alineación constructiva: cada objetivo debe poder evaluarse con el quiz o el ejercicio de la lección correspondiente._
